VwWorks 2.4.3 Release Notes

Bug Fixes:

 

DVO generated objects

á      Null pointer exception when passing a null to the equals method no correctly return false

á      Incorrect generation of user defined types where the package name was part of the data type in tee declaration – now has just the class type name

á      Re-generation of existing generic types were loosing the generic type data i.e.,.List<MyObject> was being regenerated as just List.

á      DVOÕs were not declared as Serializable and now are

 

VwSqlMgr:

á      Exceptions thrown on some JDBC drivers caused an infinite loop. Fixed

 

VwBeanToXml:

 

á      Fixed caching problem that caused incorrect attribute generation on xml documents using attribute normal form

 

VwObjectSQLMapper:

 

á      When additional graphs were added to an object mapping, existing constraints were being deleted. Fixed.

 

New Functionality:

 

VwDVOBase:

 

á      Added new method isGraphDirty that returns true if any object in the graph is dirty

á      Added new method getDirtyObjects that returns the specific objects in the graph that are dirty

á      Added new method clearGraphDirty that clears the dirty flag in all objects in the graph

 

Generated DVOÕs:

á      All generated DVOs now implement Clonable and define an equals method that operates on all objects in the graph

 

VwBeanUtils:

  • Added introspection support for enum types
  • Added new methods to test an retrieve generic type data

 

 

 

VwSchemaToJava:

  • All DVOÕs generated from XML schemas now derive from VwDVOBase

 

VwObjectSQLMapper:

  • Added new attribute superClass on the <objectProperties> tag that causes DVOs to inherit from that class. The class name must be the fully qualified name.
  • Added new attribute treatChar1AsBoolean to all mapping generation tags that generate the Boolean data type in the generated DVOs for char(1) database columns.
